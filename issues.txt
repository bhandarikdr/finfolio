# Finfolio Log Analysis - Critical Issues Report

## Issue 1: LTP Data Corruption - Flat Logic Not Enforced

### Problem Description
When the user first refreshes, correct LTPs, point changes, and change percentages are loaded. However, immediately after, **all point changes and change percentages reset to 0.00**, indicating the "flat logic" (preventing display of changes when price hasn't actually changed) is not being properly enforced.

### Root Cause Analysis

Based on the logs, the issue appears to be in the synchronization flow:

1. **Initial Sync** (07:59:41.919): Successfully scraped 323 items from Sharesansar
   ```
   [2026-06-27 07:59:41.919] [INFO] [LtpSync]: Sync successful from https://www.sharesansar.com/live-trading. Scraped 323 items.
   ```

2. **Subsequent Sync** (08:07:44.810): Successfully aggregated 748 LTP records
   ```
   [2026-06-27 08:07:48.267] [INFO] [LtpSync]: Successfully aggregated 748 LTP records
   ```

3. **The Problem**: When aggregating from multiple sources (Sharesansar + Merolagani), the app is likely:
   - Not properly matching symbols across sources
   - Using different price formats or fields
   - Overwriting previous values with defaults when source data is incomplete
   - Not maintaining previous values when new sync returns 0 items

### Bug Location (Hypothesized)
In `PortfolioRepository.kt` around the `refreshLivePrices` function (line 985 as shown in stacktrace):

```kotlin
// Current problematic implementation (hypothesis)
suspend fun refreshLivePrices() {
    val scrapedData = scrapeFromAllSources()
    
    // PROBLEM: When one source returns incomplete data, 
    // it overwrites existing values with 0.00
    scrapedData.forEach { (symbol, price) ->
        val current = getScrip(symbol)
        if (price != null && price > 0) {
            current.ltp = price
            // Calculate change from previous close
            current.pointChange = price - current.closePrice
            current.changePercent = (price - current.closePrice) / current.closePrice * 100
        } else {
            // BUG: This resets values to 0 when price is null/0
            current.pointChange = 0.0
            current.changePercent = 0.0
        }
    }
}
```

### Solution

```kotlin
class PortfolioRepository {
    private val priceCache = mutableMapOf<String, CachedPrice>()
    
    data class CachedPrice(
        val ltp: Double,
        val pointChange: Double,
        val changePercent: Double,
        val lastUpdateTime: Long
    )
    
    suspend fun refreshLivePrices() {
        val scrapedData = scrapeFromAllSources()
        val currentTime = System.currentTimeMillis()
        
        scrapedData.forEach { (symbol, newPrice) ->
            val scrip = getScrip(symbol) ?: return@forEach
            
            // 1. Validate price data
            if (newPrice != null && newPrice > 0) {
                // 2. Calculate changes based on CLOSE price, not previous LTP
                val closePrice = scrip.closePrice ?: newPrice
                val pointChange = newPrice - closePrice
                val changePercent = if (closePrice > 0) {
                    (pointChange / closePrice) * 100
                } else 0.0
                
                // 3. Update with validated data
                scrip.ltp = newPrice
                scrip.pointChange = pointChange
                scrip.changePercent = changePercent
                
                // 4. Cache the updated price
                priceCache[symbol] = CachedPrice(
                    ltp = newPrice,
                    pointChange = pointChange,
                    changePercent = changePercent,
                    lastUpdateTime = currentTime
                )
            } else {
                // 5. CRITICAL FIX: Don't reset - preserve existing values
                // Only update if we have valid new data, otherwise keep old values
                val cached = priceCache[symbol]
                if (cached != null && (currentTime - cached.lastUpdateTime) < 60000) {
                    // Keep cached values for up to 1 minute
                    scrip.ltp = cached.ltp
                    scrip.pointChange = cached.pointChange
                    scrip.changePercent = cached.changePercent
                }
                // If no cache and no new data, leave existing values unchanged
            }
        }
        
        // 6. Additional validation: Check for stale data
        validateAndCleanStaleData(currentTime)
    }
    
    private fun validateAndCleanStaleData(currentTime: Long) {
        priceCache.entries.removeIf { 
            currentTime - it.value.lastUpdateTime > 300000 // 5 minutes stale
        }
    }
}
```

### Additional Fix for Scraper Aggregation

```kotlin
class SmartAggregator {
    // Map to track which symbols have valid data from each source
    private val sourceValidity = mutableMapOf<String, MutableSet<String>>()
    
    fun aggregateScrapedData(allData: List<SourceData>): Map<String, Double> {
        val result = mutableMapOf<String, Double>()
        val validSources = mutableMapOf<String, MutableList<Double>>()
        
        allData.forEach { source ->
            source.data.forEach { (symbol, price) ->
                if (price > 0) {
                    validSources.getOrPut(symbol) { mutableListOf() }.add(price)
                    sourceValidity.getOrPut(symbol) { mutableSetOf() }.add(source.name)
                }
            }
        }
        
        // Use median or consensus price when multiple sources available
        validSources.forEach { (symbol, prices) ->
            result[symbol] = when {
                prices.size >= 3 -> calculateMedian(prices)
                prices.size == 2 -> prices.average()
                else -> prices.firstOrNull() ?: 0.0
            }
        }
        
        return result
    }
    
    private fun calculateMedian(prices: List<Double>): Double {
        val sorted = prices.sorted()
        return if (sorted.size % 2 == 0) {
            (sorted[sorted.size / 2 - 1] + sorted[sorted.size / 2]) / 2
        } else {
            sorted[sorted.size / 2]
        }
    }
}
```

## Issue 2: IPO Master List Suddenly Empty

### Problem Description
After fresh install and history load, the IPO master list is showing empty despite previous successful syncs.

### Root Cause Analysis

Looking at the logs, we can see:

1. **IPO Sync Success** (07:35:45.063): The IPO listing sync was successful
   ```
   [2026-06-27 07:35:45.063] [DEBUG] [IpoSync]: Requesting URL: https://nepalipaisa.com/api/GetIpos?stockSymbol=&pageNo=1&itemsPerPage=100&pagePerDisplay=5
   [2026-06-27 07:35:45.056] [INFO] [IpoSync]: Starting IPO Listing Sync
   ```

2. **IPO Result Mapping Success** (07:35:46.519):
   ```
   [2026-06-27 07:35:46.519] [INFO] [IpoSync]: Starting IPO Result Mapping Sync
   [2026-06-27 07:35:47.073] [DEBUG] [IpoSync]: Acquired Session Cookies: TS5c725768027=...
   ```

3. **Later Cooldown Logs** (08:15:41-08:17:09):
   ```
   [2026-06-27 08:17:09.858] [DEBUG] [IpoSync]: Skipping IPO Sync (Cooldown active)
   [2026-06-27 08:16:00.817] [DEBUG] [IpoSync]: Skipping IPO Sync (Cooldown active)
   ```

### The Issue

The IPO master list is empty because of a **database inconsistency**:

1. The sync appears successful (log shows success)
2. But the UI shows empty list
3. This indicates a **data persistence issue** - the scraped data is not being properly saved to the database

### Bug Location

In the IPO synchronization flow, likely in `IpoSync.kt` or the repository layer:

```kotlin
// Current problematic implementation (hypothesis)
class IpoSync {
    suspend fun syncIPOs() {
        val ipoList = fetchIPOList()
        val mappedResults = mapIPOResults(ipoList)
        
        // BUG: Only saves IPO results, but doesn't update the master list
        saveIPOData(mappedResults)
        // Missing: updateIpoMasterList(ipoList)
    }
```

### Solution

```kotlin
class IpoSync(
    private val ipoRepository: IpoRepository,
    private val ipoMasterListRepository: IpoMasterListRepository
) {
    
    suspend fun syncIPOs() {
        try {
            // 1. Fetch fresh data
            val ipoList = fetchIPOListFromSource()
            val mappedResults = fetchAndMapIPOResults()
            
            // 2. Validate data before saving
            if (ipoList.isNullOrEmpty()) {
                Logger.log(Level.WARN, "IpoSync", "Received empty IPO list from source")
                
                // Check if we have cached data
                val cachedList = ipoRepository.getCachedIPOs()
                if (cachedList.isNotEmpty()) {
                    Logger.log(Level.INFO, "IpoSync", "Using cached IPO data")
                    return
                }
                return
            }
            
            // 3. Save both data sets with transaction
            ipoRepository.transaction {
                // Save IPO master list
                val updatedCount = ipoMasterListRepository.saveAll(ipoList)
                Logger.log(Level.INFO, "IpoSync", "Saved $updatedCount IPOs to master list")
                
                // Save IPO results
                val resultsCount = ipoRepository.saveAll(mappedResults)
                Logger.log(Level.INFO, "IpoSync", "Saved $resultsCount IPO results")
                
                // Verify data was saved
                val verifyCount = ipoMasterListRepository.count()
                if (verifyCount == 0 && ipoList.isNotEmpty()) {
                    throw RuntimeException("Failed to save IPO master list - database write failed")
                }
            }
            
            // 4. Notify UI of data change
            notifyDataChanged()
            
        } catch (e: Exception) {
            Logger.log(Level.ERROR, "IpoSync", "IPO sync failed: ${e.message}", e)
            
            // 5. Fallback to cached data
            val cachedData = ipoRepository.getCachedIPOs()
            if (cachedData.isNotEmpty()) {
                ipoMasterListRepository.saveAll(cachedData)
                Logger.log(Level.INFO, "IpoSync", "Fallback: Restored ${cachedData.size} IPOs from cache")
            }
            
            throw e
        }
    }
    
    private fun notifyDataChanged() {
        // Use LiveData or Flow to notify observers
        _ipoMasterListFlow.value = ipoMasterListRepository.getAll()
    }
}
```

## Additional Critical Finding

### Scraper Config Validation Issue

The logs show multiple test failures for `nepalstock.com`:

```
[2026-06-27 07:18:40.729] [ERROR] [ScraperConfig]: Test Failed (Validation Error) for https://www.nepalstock.com/
[2026-06-27 07:18:30.152] [ERROR] [ScraperConfig]: Test Failed (Validation Error) for https://www.nepalstock.com/
```

**Recommendation:** The app should automatically disable/failover from `nepalstock.com` since it's consistently failing SSL validation. Add a health check mechanism:

```kotlin
class ScraperHealthManager {
    private val sourceHealth = mutableMapOf<String, HealthStatus>()
    private val MAX_CONSECUTIVE_FAILURES = 3
    
    data class HealthStatus(
        var consecutiveFailures: Int = 0,
        var isHealthy: Boolean = true,
        var lastCheckTime: Long = 0
    )
    
    fun recordFailure(source: String) {
        val status = sourceHealth.getOrPut(source) { HealthStatus() }
        status.consecutiveFailures++
        status.isHealthy = status.consecutiveFailures < MAX_CONSECUTIVE_FAILURES
        status.lastCheckTime = System.currentTimeMillis()
        
        if (!status.isHealthy) {
            Logger.log(Level.WARN, "HealthManager", "Source marked unhealthy: $source")
            // Automatically disable this source
            ScraperConfig.disableSource(source)
        }
    }
    
    fun recordSuccess(source: String) {
        val status = sourceHealth.getOrPut(source) { HealthStatus() }
        status.consecutiveFailures = 0
        status.isHealthy = true
        status.lastCheckTime = System.currentTimeMillis()
        
        // Re-enable if it was disabled
        if (!ScraperConfig.isEnabled(source)) {
            ScraperConfig.enableSource(source)
        }
    }
}
```
