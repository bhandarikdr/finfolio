## Root Causes

1. **SSL Issue**: Android 16 may have stricter certificate validation or the `nepalstock.com` certificate may be expired/self-signed.

2. **HTML Structure Change**: The DP list extraction logic likely relies on specific HTML structure that has changed on `sharesansar.com`.

3. **CDSC API/Website Access**: `cdsc.com.np` domains are either:
   - Blocked from the current network
   - Require specific headers/cookies
   - Have changed their API endpoints

### Issue 5.2: Validation Failure for CDSC URLs
**Status:** ❌ Open
**Priority:** High
**Description:** Both Meroshare web and API URLs show "URL reachable but data validation failed".

**Causes:**
1. **Missing Headers:**
   - User-Agent required
   - Accept header for JSON responses
   - Origin/Referer headers

2. **CSRF Protection:**
   - Token required for POST requests
   - Token extracted from login page

3. **Session Management:**
   - Need to maintain cookies/session
   - Use CookieManager for persistence

4. **API Response Structure Changed:**
   ```kotlin
   // Expected response vs actual response
   // Log the actual response for debugging
   Log.d("ScraperConfig", "Response: ${response.body?.string()}")
   ```

**Solution:**
```kotlin
class MeroshareScraper {
    private val client = OkHttpClient.Builder()
        .addInterceptor { chain ->
            val request = chain.request().newBuilder()
                .addHeader("User-Agent", "Mozilla/5.0 (Android 16)")
                .addHeader("Accept", "application/json, text/plain, */*")
                .addHeader("Origin", "https://meroshare.cdsc.com.np")
                .addHeader("Referer", "https://meroshare.cdsc.com.np/")
                .build()
            chain.proceed(request)
        }
        .cookieJar(CookieJar { cookies -> /* store cookies */ })
        .build()

    fun validateConnection(): Boolean {
        // Test with a simple API call
        // or validate page load success
        return try {
            val response = client.newCall(
                Request.Builder()
                    .url("https://webbackend.cdsc.com.np/api/meroShare/health")
                    .build()
            ).execute()
            response.isSuccessful
        } catch (e: Exception) {
            false
        }
    }
}
```

## STANDARDS DOCUMENTATION

### UI Standards

#### 1. **Consistent Menu Format**
```
Format: {Menu Title} ({Item Count})
Example: IPO Master (49)
```

#### 2. **Expandable Sections**
- Only one section expands at a time (accordion behavior)
- Auto-collapse other sections when expanding
- Smooth animation transitions

#### 3. **Filter/Header Layout**
```
[Left: Title/Status] [Right: Action Buttons]
Example: All (24)        [Sort] [Filter]
```

#### 4. **Family Management**
- Only display in Vault module
- Keep it security-focused
- Use clear permission flows

#### 5. **Count Display**
- Use parentheses: `(count)`
- Real-time updates when data changes
- Loading states for pending counts

---

### Data Persistence Standards

#### 1. **Watchlist**
- **Storage:** Room Database
- **Sync:** Cloud backup optional
- **Refresh:** On app start and manual pull

#### 2. **User Data**
- **Authentication:** Secure SharedPreferences with encryption
- **Family Members:** Room Database with foreign keys

#### 3. **Cache**
- **Market Data:** 5-minute TTL
- **Company Master:** 24-hour TTL
- **DP List:** 7-day TTL (with manual refresh)

---

### Scraper Configuration Standards

#### 1. **URL Testing**
- Always include headers in test
- Log full response for debugging
- Handle SPA navigation properly

#### 2. **Error Handling**
```
Error Type | Action
-----------|--------
Network    | Retry with exponential backoff (max 3)
Validation | Log HTML/Response for debugging
Timeout    | Increase timeout or use fallback source
Certificate| Add custom TrustManager or use HTTP
```

#### 3. **Fallback Strategy**
- Primary source → Secondary source → Cached data → User notification
- Log all failures with context

#### 4. **HTML Parsing**
```kotlin
// Always use multiple selectors with fallbacks
val selectors = listOf(
    "table tbody tr",
    "table tr:not(:first-child)",
    ".table-class tr"
)

selectors.forEach { selector ->
    val rows = document.select(selector)
    if (rows.isNotEmpty()) {
        // Parse rows
        return@forEach
    }
}
```

---

### Debugging Standards

#### 1. **Log Levels**
- `ERROR`: Critical failures (cert validation, data not loading)
- `WARN`: Recoverable issues (cooldown, extraction empty)
- `INFO`: Major operations (sync start/end)
- `DEBUG`: Detailed flow (URL requests, HTML structure)

#### 2. **Testing URLs**
Always test in this order:
1. Basic connectivity (ping)
2. SSL/TLS validation
3. Content type validation
4. Data extraction validation
5. Business logic validation

#### 3. **Crash Reporting**
- Include device model and Android version
- Attach last 100 log lines
- Include network state
